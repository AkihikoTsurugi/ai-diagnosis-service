# TASK-001: Google認証による会員制機能

## 1. ユーザーニーズ

- **会員限定の利用価値**: ログインしたユーザーのみがダッシュボードやプロフィールにアクセスできるようにしたい。
- **簡単なログイン**: Google アカウントを使い、パスワード管理なしで安全にログインしたい。
- **自分の情報の確認・更新**: 表示名など、サービス上で使うプロフィールを確認・編集し保存したい。
- **セキュアなセッション**: サーバーで検証可能なセッション（JWT）でログイン状態を保ちたい。
- **データの永続化**: ユーザーと OAuth 連携情報を MongoDB に保存し、再利用したい。

## 2. 仕様要件

### 2.1 認証

| 項目 | 内容 |
|------|------|
| ライブラリ | NextAuth.js v5（`next-auth@beta` / Auth.js） |
| プロバイダ | Google OAuth |
| アダプタ | `@auth/mongodb-adapter`（User / Account 等の永続化） |
| セッション戦略 | JWT（明示的に `session.strategy: "jwt"`） |
| 環境変数 | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `AUTH_SECRET` |
| 追加 env | MongoDB 接続用 `MONGODB_URI`（および任意で `MONGODB_DB`） |

### 2.2 API

| メソッド | パス | 説明 |
|----------|------|------|
| GET/POST | `/api/auth/[...nextauth]` | Auth.js 標準エンドポイント |
| GET | `/api/user/profile` | ログインユーザーのプロフィール取得 |
| PUT | `/api/user/profile` | 表示名などの更新（要認証） |

### 2.3 画面

| パス | 内容 |
|------|------|
| `/auth/signin` | Google ログインボタン。成功後 `/dashboard` へ |
| `/dashboard` | 会員向けトップ。ユーザー名・アバター・ナビ |
| `/profile` | 名前・メール・アバター表示、編集フォームと保存 |

### 2.4 ミドルウェア

- `/dashboard` および `/profile` は認証必須。
- 未ログインは `/auth/signin` へリダイレクト（可能なら `callbackUrl` を付与）。
- Edge 実行時に MongoDB ドライバを読み込まないよう、認証設定を **Edge 用（アダプタなし）** と **Node 用（アダプタあり）** に分割する。

### 2.5 データモデル（User）

MongoDB `users` コレクション（NextAuth デフォルト + 拡張）:

| フィールド | 型 | 説明 |
|------------|-----|------|
| `_id` | ObjectId | ユーザー ID（Auth.js 上は `id` として hex 文字列） |
| `name` | string | 表示名 |
| `email` | string | メール |
| `image` | string | アバター URL（Google） |
| `emailVerified` | Date \| null | メール確認日時 |
| `createdAt` | Date | 作成日時（拡張。既存ユーザーは初回 API アクセス時に補完） |
| `updatedAt` | Date | 更新日時（拡張。プロフィール更新で更新） |

メールは OAuth 由来のため、本タスクでは **編集対象外**（表示のみ）とする。

### 2.6 UI

- MUI: `Button`, `Card`, `Avatar`, `TextField` を使用。
- 既存の `ThemeRegistry` と整合するレイアウト。

## 3. 処理手順設計

### 3.1 新規・依存関係

1. `next-auth@beta`, `@auth/mongodb-adapter`, `mongodb` をインポート。
2. `lib/mongodb.ts` で `MongoClient` のシングルトン（開発時は `global` キャッシュ）。

### 3.2 認証設定の分割

1. `auth.config.ts`: `providers`, `session: jwt`, `pages.signIn`, `callbacks`（`jwt` / `session` / `authorized`）のみ。MongoDB を import しない。
2. `auth.ts`: `auth.config` を spread し `MongoDBAdapter(clientPromise)` を付与。`handlers`, `auth`, `signIn`, `signOut` をエクスポート。
3. `middleware.ts`: `NextAuth(authConfig).auth` のみ利用（Edge 安全）。

### 3.3 ルート

1. `app/api/auth/[...nextauth]/route.ts` で `handlers` の GET/POST をエクスポート。
2. `app/providers.tsx`（クライアント）で `SessionProvider` を配置し、`layout.tsx` で子要素をラップ。

### 3.4 保護ページ

1. `app/auth/signin/page.tsx`: `signIn("google", { callbackUrl: "/dashboard" })` を MUI `Button` から実行。
2. `app/dashboard/page.tsx`: サーバーで `auth()`、ユーザー名・アバター、 `/profile` へのリンク。必要なら `signOut` 用ボタン（サーバーアクション or クライアント）。
3. `app/profile/page.tsx`: `Card` + `Avatar` + 情報表示。`TextField` で名前編集。保存時は `PUT /api/user/profile` 成功後 `useSession().update({ user: { name } })` で JWT を更新。

### 3.5 プロフィール API

1. **GET**: `auth()` でセッション確認 → `users` を `ObjectId` で検索 → `createdAt` 未設定なら現在時刻で `$set` → JSON 返却。
2. **PUT**: 本文 `{ name: string }` 検証 → DB の `name`, `updatedAt` 更新 → 最新ドキュメントを返却。

### 3.6 JWT コールバック

1. ログイン時: `token.id` に `user.id`、`picture` に `user.image` を反映。
2. `trigger === "update"`: クライアントから渡した `session.user` で `token.name` / `token.picture` を更新。
3. `session` コールバック: `session.user.id` / `image` をトークンから注入。

### 3.7 型

1. `types/next-auth.d.ts` で `Session["user"]` に `id` をマージ宣言。

### 3.8 検証

1. `npm run build` が通ること。
2. `.env.local` に必要変数をセットしたうえで、サインイン〜ダッシュボード〜プロフィール更新の手動確認（本ドキュメント実装後の作業）。

## 4. 環境変数一覧（運用メモ）

| 変数名 | 必須 | 用途 |
|--------|------|------|
| `GOOGLE_CLIENT_ID` | はい | Google OAuth |
| `GOOGLE_CLIENT_SECRET` | はい | Google OAuth |
| `AUTH_SECRET` | はい | セッション暗号化（`npx auth secret` で生成可） |
| `MONGODB_URI` | はい | MongoDB 接続 URI |
| `MONGODB_DB` | 任意 | データベース名（未設定時は接続 URI のデフォルト） |
| `AUTH_URL` / `NEXTAUTH_URL` | 本番推奨 | コールバック URL 解決用 |
