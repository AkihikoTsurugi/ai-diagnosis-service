## 1. 各セクションの表示確認

ページ上から順に、主要要素が欠けず表示されること。

### 1.1 ヒーローセクション（`HeroSection`）

- [ ] キャッチコピー（h1 相当）が表示される
- [ ] サブコピーが表示される
- [ ] CTA ボタン「無料で診断スタート」が表示される
- [ ] 補足文言（登録不要・完全無料・所要時間など）が表示される
- [ ] 背景グラデーション・オーバーレイが意図どおり（崩れ・極端な色の欠けがない）
- [ ] ファーストビューで主要テキストが読める（`min-height: 100vh` 想定）

### 1.2 サービス特徴セクション（`FeaturesSection`）

- [ ] セクション見出し「SurvibeAIの特徴」が表示される
- [ ] 特徴カードが **3 枚** 並ぶ（タイトル・説明文が各カードに表示）
- [ ] 各カードにアイコンエリア（円形背景＋ MUI アイコン）が表示される

### 1.3 診断の流れセクション（`StepsSection`）

- [ ] セクション見出し「かんたん3ステップ」が表示される
- [ ] ステップが **3 つ** 表示される（番号バッジ・タイトル・説明文）
- [ ] 各ステップに MUI アイコンが表示される
- [ ] **PC幅（md 以上）** でステップ間に矢印コネクタ（`ArrowForward`）が表示される
- [ ] **スマホ幅（xs）** で矢印は非表示になり、ステップが縦に積まれる

### 1.4 下部 CTA セクション+（`CtaSection`）

- [ ] 見出し「あなたのキャリア、次の一歩を踏み出そう。」が表示される
- [ ] CTA ボタンが表示される
- [ ] 補足文言が表示される
- [ ] 背景グラデーションがヒーローと調和し、テキストが読みやすい

### 1.5 フッター（`Footer`）

- [ ] コピーライト（`© {年} SurvibeAI. All rights reserved.`）が表示される
- [ ] ダーク背景上で文字が読める（コントラスト）

## 2. レスポンシブ表示（PC / タブレット / スマホ）

各幅で **レイアウトの崩れ・はみ出し・重なり** がないこと。

### 2.1 スマホ（xs 付近）

- [ ] ヒーロー: 見出し・本文の改行・余白が自然
- [ ] 特徴: カードが **1 列** で縦スクロールしやすい
- [ ] ステップ: **縦積み**・番号・アイコンが欠けない
- [ ] 下部 CTA: ボタンがタップしやすいサイズ・位置
- [ ] 横スクロールが不要（意図しない `overflow-x` が出ない）

### 2.2 タブレット（sm〜md 付近）

- [ ] 特徴カードの列数・余白が破綻していない（md で 3 列想定）
- [ ] ヒーロー・CTA のフォントサイズが読みやすい
- [ ] ステップで md 未満のとき矢印が出ず、md 以上で矢印が出る境界付近を確認

### 2.3 PC（md 以上）

- [ ] コンテンツが `Container maxWidth` 内に収まり、左右に極端な空白のみになっていない
- [ ] 特徴 **3 列**、ステップ **横並び＋矢印** が表示される
- [ ] ホバー時（カードの浮き上がり等）が OS・ブラウザで破綻しない

## 3. CTA ボタンの動作

ヒーロー内・下部 CTA の **両方** のボタンで確認する。

- [ ] クリック / タップで反応がある（無視されない）
- [ ] **Snackbar** が画面下部付近に表示される
- [ ] メッセージが「Coming Soon! 現在準備中です 🚀」（`COMING_SOON_MESSAGE`）である
- [ ] 一定時間後に Snackbar が自動で閉じる（約 3 秒想定）
- [ ] Snackbar 表示中に再度ボタンを押した場合の挙動に問題がない（重複表示など）
- [ ] キーボード操作時: ボタンにフォーカスが当たり、Enter / Space で同様に動作する（アクセシビリティ）

> **補足（実装仕様）**: Stage1 では CTA は外部 URL への遷移ではなく、Snackbar 表示のみ。診断ページ連携後は本項目を「遷移先・パラメータ」に差し替える。

## 4. リンク遷移
