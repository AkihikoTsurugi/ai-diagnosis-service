- [ ] アプリを起動し、ログインページへ直接アクセスできる

## 1. ログインページの表示（`/auth/signin`）

- [ ] `GET /auth/signin` が **200** で開ける
- [ ] ページタイトル相当の見出し（例: 「ログイン」）が表示される
- [ ] 会員エース利用に関する説明文言が表示される
- [ ] 「Google でログイン」ボタン（または同等のラベル）が **1 つ** 明確に表示される
- [ ] レイアウトが崩れていない（カード中央寄せ、スマホ幅でもはみ出しがない）

> **補足（実装仕様）**: サインイン画面は `app/auth/signin/page.tsx`。NextAuth の `signIn` ページは `auth.config.ts` の `pages.signIn` で `/auth/signin` に設定。

---

## 2. Google ログインボタンの動作

- [ ] 「Google でログイン」をクリックすると Google の認証画面（アカウント選択またはログイン）へ遷移する
- [ ] 認証を **キャンセル** した場合、適切にアプリ側へ戻る（エラー表示またはサインイン画面に留まるなど、仕様どおり）
- [ ] **別タブ・別ウィンドウ** で二重に押した場合の挙動に致命的な不整合がない（可能なら確認）

> **補足（実装仕様）**: `signIn("google", { callbackUrl: "/dashboard" })` により、成功時の既定遷移先はダッシュボード。

---

## 3. ログイン成功後のリダイレクト

- [ ] Google 認証が成功すると **`/dashboard` に遷移する**（または意図した `callbackUrl` へ遷移する）
- [ ] 遷移後、ダッシュボードで **ログイン済み** の状態としてコンテンツが表示される（未ログイン用のメッセージだけにならない）
- [ ] URL バーが `/dashboard` である

---

## 4. ダッシュボードの表示（`/dashboard`）

**前提: ログイン済み**

- [ ] `/dashboard` が **200** で表示される
- [ ] 会員向けの画面として主要ブロック（ナビ・見出し・コンテンツ等）が欠けず表示される
- [ ] ダッシュボードから **プロフィール** への導線（リンク）があり、クリックで `/profile` に遷移できる

**セッションなしでのアクセス（項目 8 と併せても可）**

- [ ] 未ログインで `/dashboard` にアクセスした場合、仕様どおり **サインインへ誘導** される（リダイレクトまたは表示のいずれか）

---

## 5. プロフィールページの表示（`/profile`）

**前提: ログイン済み**

- [ ] `GET /profile` が **200** で開ける
- [ ] ユーザー情報（表示名・メール等、実装されている項目）が **取得結果どおり** に表示される
- [ ] ローディング中・エラー時の表示があれば、過去に失敗したときと区別できる（任意）
- [ ] ダッシュボードへ戻る導線があれば動作する

---

## 6. プロフィール編集の動作

**前提: ログイン済み、編集 UI がある場合**

- [ ] 編集可能なフィールドが画面上に存在する
- [ ] 値を変更して **保存** すると成功フィードバック（トースト・メッセージ等）が出る、または画面に反映される
- [ ] ページを **再読み込み** しても、保存した内容が維持される（API `PUT` / `PATCH` と DB の組み合わせが正しいことの確認）
- [ ] 不正な入力・必須未入力がある場合、バリデーションエラーが表示される（実装されている場合）

> API は `app/profile` 周辺の `fetch("/api/user/profile", …)` に依存。バックエンド変更時は本項目を必ず再確認すること。

---

## 7. ログアウト機能

- [ ] ログアウト用の UI（ボタン・メニュー）が見つかり、クリックできる
- [ ] ログアウト後、**会員向けページにアクセスできない**（または未ログイン状態の表示になる）
- [ ] ログアウト後に `/auth/signin` へ誘導される、またはそこから再度ログインできる
- [ ] ログアウト直後に「戻る」で保護ページに入れない（セッションが無効化されている）

---

## 8. 未ログイン時のリダイレクト（保護ルート）

- [ ] **未ログイン** の状態（シークレットウィンドウまたはストレージ削除）で `/dashboard` にアクセスすると `/auth/signin`（または設定されたサインインページ）へ **リダイレクト** される
- [ ] 同様に `/profile` に直接アクセスしても未ログインではサインインへ誘導される（アプリ実装および `middleware.ts` の matcher と整合）

> **補足（実装仕様）**: `middleware.ts` は `/dashboard/:path*` と `/profile/:path*` をマッチ。`auth.config.ts` の `authorized` でログイン状態を判定。

---

## 9. セッションの維持

- [ ] ログイン後、**同一ブラウザで別タブ** を開き、`/dashboard` または `/profile` がログイン済みで表示される
- [ ] **ページリロード（F5）** 後もログイン状態が維持される
- [ ] **短時間のブラウザの再起動** 後も、Cookie / セッション期限内在庫でログインが維持される（期待する保持時間はプロジェクト方針に合わせて記載）
- [ ] セッション期限切れ・サーバー再起動後の挙動が仕様どおりである（分かる範囲で確認）
