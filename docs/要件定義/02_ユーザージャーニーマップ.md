# ユーザージャーニーマップ

## 概要

LP訪問からCTAクリック（診断開始）までのユーザー体験を、ペルソナごとにマッピングする。

---

## 共通ジャーニーフロー

```
[認知] → [LP流入] → [ヒーロー閲覧] → [特徴理解] → [流れ確認] → [CTA押下]
```

### フェーズ定義

| フェーズ | ユーザーの状態 | 対応するLP要素 |
|----------|-------------|---------------|
| 1. 認知 | サービスの存在を知る | （LP外：広告、SNS、検索） |
| 2. 興味 | LPに訪問し、第一印象を受ける | ヒーローセクション |
| 3. 理解 | サービスの価値を理解する | サービス特徴セクション |
| 4. 納得 | 利用のハードルが低いと感じる | 診断の流れセクション |
| 5. 行動 | 診断を開始する | CTAボタン |

## ペルソナ別ジャーニー詳細

### ペルソナ1：田中 美咲（22歳・就活生）

| フェーズ | 行動 | 思考 | 感情 | タッチポイント |
|----------|------|------|------|-------------|
| 認知 | Xで友人がシェアした診断結果を見る | 「みんなやってるやつだ、面白そう」 | 好奇心 | X（シェア投稿） |
| 興味 | リンクをタップしてLPを開く | 「5問だけ？すぐ終わりそう」 | 安心感 | ヒーローセクション |
| 理解 | 特徴を流し読みする | 「AIが分析してくれるなら自己分析より楽かも」 | 期待 | サービス特徴 |
| 納得 | 3ステップの流れを確認 | 「本当に簡単そう、無料だし」 | 後押し | 診断の流れ |
| 行動 | CTAボタンをタップ | 「とりあえずやってみよう」 | ワクワク | CTAボタン |

**ジャーニー所要時間：約30秒〜1分**

**離脱リスクポイント：**
- ヒーローセクションで「またよくある自己分析か」と感じて離脱
- ページの読み込みが遅いとすぐ離脱

**対策：**
- キャッチコピーで従来の自己分析との差別化を明示
- ファーストビューにCTAを配置し、即座にアクションできる導線
- ページの軽量化（LCP 2.5秒以内）

## ジャーニー上のKPI設計

| フェーズ | KPI | 目標値 |
|----------|-----|--------|
| 認知→興味 | LP流入数 | - |
| 興味→理解 | スクロール率（特徴セクション到達） | 60%以上 |
| 理解→納得 | スクロール率（診断の流れ到達） | 40%以上 |
| 納得→行動 | CTAクリック率 | 15%以上 |
| 全体 | 直帰率 | 50%以下 |
| 全体 | 平均滞在時間 | 45秒以上 |

## LP設計への示唆

### ヒーローセクション（興味フェーズ）

- **必須要素：** キャッチコピー、サブコピー、CTAボタン
- **設計方針：** 「5問」「無料」「AI」の3つのキーワードを瞬時に伝える
- **ペルソナ横断の訴求軸：** 手軽さ × 精度の高さ

### サービス特徴セクション（理解フェーズ）

- **必須要素：** 3つの特徴を視覚的にわかりやすく
- **設計方針：** 各ペルソナの疑念（精度、中立性、深さ）に応える内容
- **推奨特徴案：**
  1. AIがあなた専用のロードマップを作成
  2. 転職・就活に偏らない中立的な分析
  3. たった5問、3分で完了

### 診断の流れセクション（納得フェーズ）

- **必須要素：** 3ステップのビジュアルフロー
- **設計方針：** ハードルの低さを視覚的に示す
- **推奨ステップ案：**
  1. 5つの質問に回答（約1分）
  2. AIがあなたの回答を分析
  3. パーソナライズされたキャリアロードマップを受け取る

### 下部CTAセクション（行動フェーズ）

- **必須要素：** CTAボタン、補足コピー
- **設計方針：** 最後の一押しとなる安心材料を添える
- **推奨コピー例：** 「登録不要・完全無料・たった5問」
