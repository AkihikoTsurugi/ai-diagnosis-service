# ユーザーニーズ洗い出し

## 概要

ペルソナ分析とユーザージャーニーから導出されたユーザーニーズを、機能ニーズ・情緒ニーズ・社会的ニーズの3階層で整理する。

---

## 1. 機能ニーズ（Functional Needs）

サービスに対して「できてほしいこと」の具体的な要求。

### 1.1 診断体験に関するニーズ

| ID | ニーズ | 対象ペルソナ | 優先度 |
|----|--------|------------|--------|
| F-01 | 短時間（5問以内）で診断が完了すること | 全ペルソナ | 最高 |
| F-02 | 会員登録・ログイン不要で診断できること | 全ペルソナ | 最高 |
| F-03 | 無料で利用できること | 全ペルソナ | 最高 |
| F-04 | スマートフォンで快適に操作できること | 田中美咲、佐藤健太 | 最高 |
| F-05 | 質問の意図がわかりやすいこと | 全ペルソナ | 高 |

### 1.2 診断結果に関するニーズ

| ID | ニーズ | 対象ペルソナ | 優先度 |
|----|--------|------------|--------|
| F-06 | 適性のある職種・業界が具体的に提示されること | 田中美咲 | 最高 |
| F-07 | キャリアロードマップ（時系列の行動計画）が得られること | 全ペルソナ | 最高 |
| F-08 | 複数のキャリアパスが比較できること | 山田彩 | 高 |
| F-09 | 転職/残留を含めた中立的な提案であること | 佐藤健太 | 高 |
| F-10 | 診断結果を保存・シェアできること | 田中美咲 | 中 |

### 1.3 LP（Stage1）に関するニーズ

| ID | ニーズ | 対象ペルソナ | 優先度 |
|----|--------|------------|--------|
| F-11 | サービスの内容が一目で理解できること | 全ペルソナ | 最高 |
| F-12 | 診断開始までの導線が明確であること | 全ペルソナ | 最高 |
| F-13 | ページの読み込みが速いこと | 田中美咲 | 高 |
| F-14 | 診断の流れが事前にわかること | 山田彩、佐藤健太 | 高 |

## 2. 情緒ニーズ（Emotional Needs）

サービス利用時に「感じたいこと」の心理的な要求。

| ID | ニーズ | 対象ペルソナ | LP上での対応方針 |
|----|--------|------------|----------------|
| E-01 | キャリアの不安が軽減される安心感 | 全ペルソナ | 共感的なサブコピーで「悩んでいるのはあなただけじゃない」を伝える |
| E-02 | 自分の可能性が広がるワクワク感 | 田中美咲 | ヒーローセクションのビジュアル・コピーで未来志向を演出 |
| E-03 | 客観的に自分を知れる知的満足感 | 山田彩 | AI分析のロジック・信頼性を特徴セクションで訴求 |
| E-04 | 判断材料を得られた達成感 | 佐藤健太 | 診断結果のイメージを特徴セクションで示す |
| E-05 | 「このサービスは信頼できる」という安心感 | 佐藤健太、山田彩 | 清潔感のあるデザイン、AI技術の説明 |
| E-06 | 手軽に始められるカジュアルさ | 田中美咲 | 「たった5問」「1分で完了」の強調 |

---

## 3. 社会的ニーズ（Social Needs）

他者との関係性の中で生まれる要求。

| ID | ニーズ | 対象ペルソナ | LP上での対応方針 |
|----|--------|------------|----------------|
| S-01 | 診断結果をSNSでシェアして話題にしたい | 田中美咲 | （Stage2以降）OGP対応のシェア機能 |
| S-02 | 友人・同僚にも勧められるサービスであること | 全ペルソナ | 洗練されたデザインとUXで紹介のハードルを下げる |
| S-03 | 同世代のキャリア傾向と比較したい | 田中美咲、佐藤健太 | （Stage2以降）統計データの表示 |
| S-04 | パートナーや家族にキャリアプランを共有したい | 山田彩 | （Stage2以降）PDF出力・共有機能 |

---

## 4. 隠れたニーズ（Latent Needs）

ユーザー自身が明確に言語化していないが、存在する潜在的な要求。

| ID | ニーズ | 洞察 | LP上での対応方針 |
|----|--------|------|----------------|
| L-01 | 「診断する」という行為自体による自己肯定感 | 行動を起こしたこと自体がキャリア不安の解消につながる | CTAコピーで「最初の一歩を踏み出す」ニュアンスを持たせる |
| L-02 | 自分の悩みを言語化してほしい | 漠然とした不安を構造化してほしいという欲求 | ヒーローのサブコピーで悩みを代弁する |
| L-03 | 「正解」を教えてほしいのではなく「選択肢」を整理してほしい | 最終的な決断は自分でしたいが、情報整理は外部に委ねたい | 特徴で「ロードマップを提示」「選択はあなた自身」のトーン |
| L-04 | 自分に投資する価値があると確認したい | キャリアに時間をかけること自体に罪悪感がある層への肯定 | 「たった1分の投資で、あなたの未来が見える」のような表現 |

## 5. ニーズの優先度マトリクス

### Stage1（LP）で対応すべきニーズ

```
             重要度 高
              │
    F-01,02,03│F-06,07,11,12
    F-04,13   │E-01,02,05
              │L-01,02
   ───────────┼───────────
    S-01,03   │F-08,09,10
    S-04      │E-03,04,06
              │F-05,14
              │L-03,04
              │
             重要度 低
     緊急度 低        緊急度 高
```

### Stage1 LP 対応ニーズ一覧（優先順）

1. **必須対応（MVP）**
   - F-01: 5問で完了する手軽さの訴求
   - F-02: 登録不要の明示
   - F-03: 無料の明示
   - F-04: モバイルファーストのレスポンシブ設計
   - F-11: サービス内容の即時理解
   - F-12: CTA導線の明確化
   - E-01: 安心感の醸成

2. **重要対応**
   - F-06, F-07: 診断結果イメージの提示
   - F-13: ページパフォーマンス最適化
   - F-14: 診断フローの可視化
   - E-02, E-05: ワクワク感と信頼感の両立
   - L-01, L-02: 潜在ニーズへの訴求コピー

3. **Stage2以降に延期**
   - F-08, F-09, F-10: 診断結果の詳細機能
   - S-01〜S-04: ソーシャル機能全般
   - E-03, E-04: 知的満足感・達成感（診断結果で対応）
